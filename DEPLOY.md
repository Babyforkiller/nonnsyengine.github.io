# Инструкция по загрузке на GitHub Pages

Эти инструкции нужны только при **обновлении** сайта с локального проекта.

## Шаг 1: Запустите скрипт подготовки

В папке проекта (где лежит `web/`):

```bash
python web/prepare_github_pages.py
```

Или: `python3 web/prepare_github_pages.py`

## Шаг 2: Скопируйте файлы в репозиторий

Содержимое папки `github-pages/` загрузите в **корень** репозитория (замените старые файлы).

## Шаг 3: Настройки GitHub Pages

**Settings** → **Pages** → **Source**: ветка `main`, папка **/ (root)** → **Save**.

## Шаг 4: Проверка

Через 1–2 минуты сайт доступен по адресу репозитория, например:  
`https://babyforkiller.github.io/nonnsyengine.github.io/`

---

Файл `.nojekyll` в корне обязателен для корректной работы.
